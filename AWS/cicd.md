# AWS Codeサービス学習記録 (CI/CD)

## 実施したこと
- GitHubリポジトリの作成
- CodeBuild, CodeDeploy, CodePipelineの設定
- CI/CDパイプラインの構築と実行

## CodeBuild
- **役割**: ソースコードをコンパイルし、テストを実行し、デプロイ可能なパッケージを作成する。
- **重要設定**: `buildspec.yml`
    - リポジトリのルートに配置する「ビルドへの指示書」。
    - インストールするランタイムや、実行するコマンド、出力するアーティファクト（成果物）を記述する。

## CodeDeploy
- **役割**: EC2などのサーバーにアプリケーションを自動配置する。
- **重要設定**: `appspec.yml`
    - リポジトリのルートに配置する「デプロイへの指示書」。
    - どのファイルをサーバーのどこに置くか、配置後にどのスクリプト（Apache再起動など）を実行するかを記述する。
- **前提条件**:
    - EC2にIAMロール（S3アクセス権限）が必要。
    - EC2内でCodeDeploy Agentが起動している必要がある。

## CodePipeline
- **役割**: Source(GitHub) → Build → Deploy の流れを管理する司令塔。
- GitHubへのPushを検知して自動的にプロセスを開始する。

## 現時点での理解レベル
- CodePipelineが全体の「司令塔」で、BuildとDeployに指示を出しているイメージ。
- コンソール上の設定だけでなく、リポジトリに入れる「2つのYAMLファイル（buildspec.yml, appspec.yml）」が動作の鍵を握っていると理解。
- CodeDeployは単にファイルを置くだけでなく、エージェントと連携してスクリプト実行までやってくれる。
