# AWSネットワーク・ロケーション学習記録

## 学習概要
- エッジロケーション、ローカルゾーン、Outpostsの違い
- ルートテーブルの「ターゲット」と「送信先」
- ステートフルとステートレス
- ELBのL4 (NLB) と L7 (ALB)

## AWSのロケーション概念
- **エッジロケーション**:
    - コンテンツ配信（CDN）のための「出張所」。
    - CloudFrontのキャッシュなどに使われる。世界中に多数ある。
- **ローカルゾーン**:
    - 特定の都市（大都市）にある「ミニ・リージョン」。
    - 地理的な遅延（レイテンシー）を極限まで減らしたいアプリ（ゲームなど）向け。
- **Outposts**:
    - 「自社データセンター」にAWSのラックを置くサービス。
    - 物理的に一番近い。工場制御やデータレジデンシー要件向け。

## ネットワーク基礎
- **ルートテーブル**:
    - **送信先 (Destination)**: 行きたい場所（住所）。IPアドレスで指定（例: `0.0.0.0/0`）。
    - **ターゲット (Target)**: 手段（乗り物）。AWSリソースで指定（例: `igw-xxx`）。
- **ステートフル vs ステートレス**:
    - **ステートフル (Security Group)**: 「行き」を許可すれば「帰り」も自動許可（顔パス）。設定が楽。
    - **ステートレス (Network ACL)**: 「行き」と「帰り」を別々にチェック。設定が面倒だが厳密。

## ELB (Elastic Load Balancer)
- **L4 (NLB)**:
    - トランスポート層。IPとポート番号だけ見て振り分ける。
    - **超高速**。中身を見ない。
- **L7 (ALB)**:
    - アプリケーション層。URLやヘッダーの中身を見て振り分ける。
    - **高機能**。パスベースルーティングなどが可能。Webアプリ向け。

## 現時点での理解レベル
- ターゲット＝「手段（乗り物）」という理解でスッキリした。
- ステートフルは「ドアマン（顔パス）」、ステートレスは「空港の保安検査（毎回チェック）」というイメージ。
- Webアプリなら基本はALB（L7）で、速度重視ならNLB（L4）。
