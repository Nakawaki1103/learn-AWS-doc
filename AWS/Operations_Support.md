# AWS運用・サポートツール学習記録

## 学習概要
- Systems Manager (SSM) の主要機能
- 管理・サポート系サービス（License Manager, Health, Trusted Advisor）

## Systems Manager (SSM)
- **役割**: AWS運用のための「十徳ナイフ（ツール詰め合わせ）」。
- **主要機能**:
    - **Session Manager**: SSH不要、ポート開放不要でブラウザから安全にEC2接続できる（踏み台サーバー不要論の主役）。
    - **Parameter Store**: パスワードや設定値を安全に保存・管理する。
    - **Run Command**: サーバーにログインせず、複数台一括でコマンド実行。
    - **Patch Manager**: OSのアップデート（パッチ適用）を自動化。

## 管理・サポート系サービス
- **AWS License Manager**:
    - 「ライセンス管理者」。WindowsやOracleなどの商用ライセンスの使用状況（CPUコア数など）を管理し、違反（使いすぎ）をブロックする。BYOLで必須。
- **AWS Health**:
    - 「運行情報掲示板」。AWS全体の障害情報や、自分のリソースに関わるメンテナンス予定を通知する。
- **AWS Trusted Advisor**:
    - 「お節介なコンサルタント」。コスト、セキュリティ、耐障害性などの観点から、AWS環境を自動チェックして改善案（もっと安くできる、危ない設定がある等）を提示する。

## 現時点での理解レベル
- SSMは単一のサービスというより「運用ツールの集合体」と理解。特にSession Managerはセキュリティ的に強力。
- Trusted Advisorは「AWS公認のコンサル」で、ベストプラクティスに沿っているかチェックしてくれる存在。
