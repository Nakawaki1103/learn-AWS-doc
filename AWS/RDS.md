# RDS学習記録

## 実施したこと
- Primary-Standby構成を作成する
- RDSインスタンスの詳細の確認とパラメータを変更する
- EC2インスタンスからRDSインスタンスに接続
- RDSのスナップショットの取得とリストア
- RDSのフェイルオーバー機能を試す

## RDSとは
- フルマネージドなリレーショナルデータベースサービス

## Primary-Standby構成とは
- Multi-AZ配置とも呼ばれ可用性向上に繋がる
	- エンドポイントで接続→レプリケーションで同期→接続に障害が起きた場合はフェイルオーバーでサービス停止を回避（向き先が自動的に変わる）
- リードレプリカ、参照用（アプリケーションで使われるコマンドの多くはselectのため）のRDSを用意し、書き込む用のRDSと分けることで負荷を分散させる
