# AWSデプロイ・データ分析サービス学習記録

## 学習概要
- アプリケーションデプロイ自動化 (Proton, Elastic Beanstalk)
- S3データ活用・分析 (Athena, S3 Select)

## アプリケーションデプロイ自動化

### AWS Proton
- **役割**: コンテナ・サーバーレス開発のための「インフラ + CI/CD」テンプレート管理サービス。
- **特徴**:
    - インフラ担当が「環境」と「サービス」のテンプレート（CI/CD設定含む）を作成。
    - 開発者はテンプレートを選んでリポジトリを連携するだけで、自動で環境が整う。
- **Service Catalogとの違い**:
    - Protonは「CI/CDパイプライン」まで含めて管理する。
    - テンプレート更新時に、既存のリソースへも一括で変更を適用できる（バージョン管理が強力）。

### Elastic Beanstalk
- **役割**: Webアプリ（Java, PHP, Pythonなど）の「全自動構築サービス」。
- **特徴**:
    - コード（WARファイルなど）をアップロードするだけで、EC2, ELB, Auto Scaling, RDSなどのインフラ一式を自動構築する。
    - 「定食屋」のイメージ。構成は決まっているが、サイズなどは調整可能。
- **使い所**: 既存のEC2ベースのWebアプリを、手間をかけずにクラウドで動かしたい時。

## S3データ活用・分析

### Amazon Athena
- **役割**: S3上のビッグデータ（ログなど）に対して、標準SQLで分析できるサーバーレスサービス。
- **特徴**:
    - インフラ管理不要。S3にファイルを置いてSQLを投げるだけ。
    - 大規模な集計や結合（JOIN）が得意。
    - 料金はスキャンしたデータ量従量課金。

### S3 Select
- **役割**: S3上の「1つのファイル」から必要なデータだけを抽出してダウンロードする機能。
- **特徴**:
    - 「分析」というよりは「転送量の削減・高速化」が目的。
    - アプリ側でフィルタリングするのではなく、AWS側で絞り込んでから転送することで効率化する。
    - 複雑なSQLは使えない（シンプルなSELECTのみ）。

## 現時点での理解レベル
- Protonは「CI/CD付きのService Catalog」で、モダンなコンテナ開発向け。
- Beanstalkは「EC2版の全自動構築」で、古き良きWebアプリ向け。
- Athenaは「図書館全体の検索（分析）」、S3 Selectは「1冊の本の目次検索（ダウンロード効率化）」という違いで理解。
