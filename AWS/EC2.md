# EC2学習記録

## 実施したこと
- インスタンスの起動
- CloudShellでのインスタンス接続とミドルウェアをインストール
- AMIを取得し、インスタンスを起動する
- ElasticIPアドレスを使ってIPアドレスを固定する

## EC2とは
- クラウド上で仮想サーバーを構築、実行できるサービス
- 必要なときに必要なリソースを確保でき、自由度が高い
- 柔軟なスペック変更か可能

## EBSとは
- EC2インスタンスにアタッチして使われるストレージサービス

## ElasticIPとパブリックIP
- パブリックIPはインスタンスを停止、終了すると解放され再起動時に新しいパブリックIPが割り当てられる
- ElasticIPはインスタンスに永続的に割り当てることができ、ユーザーに対して割り当てられる。特定のインスタンスに紐づけることが可能←便利！

　- RDSというサービスがあるから実施しないだけでEC2にデータベース機能を持つミドルウェアをインストールすればDBとして動く。というかRDSはそういうサービス？
- AMIはテンプレートのようなイメージ、AMIを作成しておくことで同一設定のEC2インスタンスを複数個起動できる
